error_dialog(message) Display a popup with the given error message.
Click to see example
confirm_dialog(message, handler) Ask confirmation from the user: display a popup with the given message, and buttons Yes and No.
handler is called with a parameter set to true if the user answers Yes, or set to false if the user answers No or close the popup without answering.
Click to see example
input_dialog(
  icon,
  title,
  message,
  default_value,
  max_length,
  validation_handler,
  ok_handler
) 		Display a popup, asking to enter a value.
icon, titleThe popup will have this title and icon in its title bar
messageText displayed inside the popup, before the area where the user can enter the requested value
default_valuepre-fill the value
max_lengthmaximum number of characters the value can have
validation_handler function called each time the user change the value, to verify if the value is correct or not.
It will receive one paramter: the value.
It should return:
  • a string: in case of error, this is the error message that will be displayed to the user
  • null in case the given value is correct
ok_handlerIf the user clicks on the Ok button, ok_handler will be called with the value entered by the user as parameter, and the popup will be closed.
Click to see example
multiple_input_dialog(icon,title,inputs,ok_handler) Similar as input_dialog but supports several input fields, given in inputs.
inputs is an array of object having the following attributes:
  • message: text to be displayed in front of the input field
  • default_value: the initial value the input field will be pre-filled with
  • max_length: maximum number of character that the input field can contain
  • validation_handler: function to validate the field: returns null if the field is correct, or a string containing the error message if the field is not correct.
